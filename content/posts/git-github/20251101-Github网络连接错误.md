本文记录了问题原因及详细解决步骤，包括如何获取代理 IP 和端口、配置 Git 全局代理（git config --global http.proxy），以及验证配置是否生效，帮助你快速修复 Git 无法访问 GitHub 的问题

<!--more-->

## 前言

今天我更换了新的网络代理，在项目中执行 `git pull` 时却突然失败，报错信息如下

> fatal: unable to access 'https://github.com/xxx/xxxxxx.git/': Failed to connect to x.x.x.x port xxxx after xxxx ms: Couldn't connect to server

这通常是由于 代理未正确配置 或 代理端口不通 导致的。以下是详细解决步骤。

## 解决步骤

### Step 1：确认代理 IP 与端口

![截图1](https://img.mgrowup.com/imgs/2025/11/0ecfe4390cd8eb53.png "图1 设置-代理-编辑")

![截图2](https://img.mgrowup.com/imgs/2025/11/f94f9e0e68b976c4.png "图2 拷贝代理IP地址和端口号")

- IP代理地址：127.0.0.1（以自己电脑的为准）
- 端口号：7078（以自己电脑的为准）

### Step 2：设置全局代理

（2）打开 terminal ，输入以下命令设置代理：

```bash
git config --global http.proxy http://127.0.0.1:7078
```

设置完成后，可通过以下命令检查，确认 http.proxy 字段已正确显示

```bash
git config --global -l
```

### Step 3：重新执行命令

重新打开一个新的 Terminal 窗口，执行 `git pull` ，若代理配置正确，此时应可正常连接到 GitHub 并拉取代码。

## 注意事项

1. 如果刚刚切换了代理工具，可能在系统设置中显示“使用代理服务器”为【关】（见图1），建议重启电脑后再试。如果你仅想在特定项目中生效，可以去掉 `--global` 参数，仅在当前仓库配置：

```bash
git config http.proxy http://127.0.0.1:7078
```
